Sticky floating box

06 Nov 2011 by vsync

This is a jQuery plugin I wrote back in 2009 (improving it regularly since). It’s highly useful for side menus that you wish to float or to be “stuck” on screen as the user scrolls around your website. It’s funny because I still get fan mail for making this plugin, so I recently re-released it with extra features, so check it out.

Note: I have completely re-built this plugin as of October 25, 2012
It is much more versatile now, allowing separate settings for each floated element, updating it’s settings on the fly or destroying it. You should really check out the demo page.


I would call this an enhanced position fixed. This plugin makes it possible to have a fixed position element that is relative to it’s parent. A normal fixed positioned element would be “out of context” and is very difficult to use in the most common situations with fluid designs. This plugin solves that problem with as little code as I could possible get it so it will run in the most optimized way, while also allow you to customize it in many important ways which might suit you best.


CSS3 transitions support

My suggestion to you is to always use CSS transition if you want to achieve a smooth floating effect to your floated element. CSS transitions are using the GPU of your graphic chip, and are not calculated using javascript (js is slow for rapid calculations), so the end result would look much better. I wrote this plugin way back when CSS was really primitive, but now I would suggest you do the following: Create a class for your floated element with the desired transition, something like this: 


.transition200{ transition:200ms; -webkit-transition:200ms; -o-transition:200ms; }
And then run the plugin with ‘cssTransition‘ setting set to ‘true’. This will make sure to use your CSS3 transition, and if the browser doesn’t support it, use the slower javascript transition (IE <10 for example). Now, I didn't want the plugin to actually do this for you automatically because it's not a clean solution, setting hard coded styles on each floated elements and if would be hard to create more than basic transitions using a cubic-bezier easing algorithm.
If you don’t want to write any CSS but want this plugin to use CSS3 transitions automatically if possible, I would suggest using one of these solutions below (load them before the stickyfloat plugin):

jquery.transition.js
jQuery-Animate-Enhanced


Benefits

  • Can handle many floated elements, each with it’s own settings
  • Floated elements’ movement can be bound inside their parents’ area
  • Uses Native javascript easing, but using CSS3 transitions would be the optimum (not for IE)
  • Has many settings, such as stick to the bottom or animation style
  • Code is very maintainable and higly efficient

CSS setup notice

Make sure you set up your CSS manually so the floating element will have position:absolute and it’s parent will be position:relative, or else the floating wouldn’t work.

Example – initialize

jQuery('.menu').stickyfloat( {duration: 400} );

Example – update (change settings for a specific element)

jQuery('.menu').stickyfloat('update',{duration : 0, stickToBottom:true });

Example – destroy (no more floating)

jQuery('.menu').stickyfloat('destroy');

Parameters

  • duration (200) The duration of the animation
  • startOffset (number) The amount of scroll offset after the animations kicks in
  • offsetY (number) The offset from the top when the object is animated
  • lockBottom (true) Set to false if you don’t want your floating box to stop at parent’s bottom
  • stickToBottom (false) Make whatever that is “floating” sticl top the bottom instead to the top
  • delay (0) Delay in milliseconds until the animnations starts
  • easing (linear) Easing function (jQuery has by default only ‘swing’ & ‘linear’)
  • cssTransition (false) if you wish to manualy set your own CSS3 transion styles on the floated element, flag this setting as ‘true’
